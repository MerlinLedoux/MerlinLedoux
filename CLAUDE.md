# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository purpose

This is a GitHub profile README repository. The repo name matches the GitHub username (`MerlinLedoux/MerlinLedoux`), which is a special repository type where `README.md` is rendered directly on the user's GitHub profile page.

The profile is built from generated SVG assets (ASCII portrait, info card, contribution heatmap) rather than hand-authored graphics тАФ see Architecture below.

## Architecture

`README.md` embeds three generated SVGs side by side in a table (portrait width 370, info card width 490, so they line up):

- **`avi-ascii.svg`** тАФ a monochrome ASCII-art portrait that "types" itself in via SMIL animation (GitHub runs SMIL in embedded `<img>` SVGs, not JS). Generated by `scripts/make_ascii_svg.py` from `source-prepped.png`.
- **`info-card.svg`** тАФ a neofetch-style key/value panel (education, stack, tools). Static content is hand-authored in the `ROWS` list at the top of `scripts/make_info_card.py`; rows fade in on a stagger.
- **`contrib-heatmap.svg`** тАФ a GitHub-style contribution grid with real streak/stats data, rendered by `scripts/render_heatmap_svg.py` from `data/contributions.json`.

### Generation pipeline

1. `scripts/prep_photo.py <photo>` тАФ removes background (rembg) + boosts local contrast (CLAHE), composites onto white тЖТ `source-prepped.png`. Local, one-off, only needed when the source photo changes.
2. `scripts/make_ascii_svg.py` тАФ samples `source-prepped.png` into a character grid тЖТ `avi-ascii.svg`. Tunable constants at the top (`CONTRAST`, `GAMMA`, `WHITE_FLOOR`, etc). Set `STATIC=1` env var to emit the frozen final frame instead of the animated version (useful for previews).
3. `scripts/make_info_card.py` тАФ edit the `ROWS` list (and `HOST`) directly in the script, then run it to regenerate `info-card.svg`. `STATIC=1` works the same way here.
4. `scripts/fetch_contributions.py` тАФ scrapes GitHub's public unauthenticated contributions endpoint (no token needed) for `GH_PROFILE_USER`, writes `data/contributions.json` (days, streaks, monthly totals).
5. `scripts/render_heatmap_svg.py` тАФ turns `data/contributions.json` into `contrib-heatmap.svg`.

Steps 4тАУ5 run daily via `.github/workflows/update-profile-art.yml` (cron `17 3 * * *` + manual `workflow_dispatch`), which commits the refreshed JSON/SVG back to the repo with zero auth beyond the default `GITHUB_TOKEN`. Steps 1тАУ3 are local-only and not automated (they depend on a source photo / hand-authored bio content).

### Dependencies

- `requirements-local.txt` тАФ needed only for the local one-time image steps (pillow, numpy, opencv-python, rembg, onnxruntime).
- `scripts/requirements.txt` тАФ needed by the daily CI workflow (requests, beautifulsoup4 only).

`DSCF3891.JPG` (source photo) and `source-prepped.png` (intermediate) are excluded via `.gitignore` тАФ they're working files for regenerating the portrait, not part of the published profile.

## Common tasks

- Re-tune the portrait: edit constants in `scripts/make_ascii_svg.py`, then `python scripts/make_ascii_svg.py`.
- Edit the info panel: edit `ROWS`/`HOST` in `scripts/make_info_card.py`, then `python scripts/make_info_card.py`.
- Force a contribution graph refresh: run the `update-profile-art.yml` workflow manually (`workflow_dispatch`), or run `scripts/fetch_contributions.py` + `scripts/render_heatmap_svg.py` locally with `GH_PROFILE_USER` set.
